# 基于HyperLegder的区块链功能验证的Demo程序开发

## 功能需求描述

每个钱包里是这样
- 有不同的资产， 每个资产有两种不同的形态， 每种形态有一个可以为负值的余额
- 新资产默认余额是0

例如
- 钱包1
  - RMB, digital, 0, cash, 50
  - USD, digital, -200, cash, 60

- 钱包2，
  - RMB, digital, 0, cash, 20

钱包之间能转账， 转账额度可能为负数

例如
- 钱包1 转给 钱包2  RMB, digital, 100，  20190102-132221
- 钱包2 转给 钱包1  JPY, digital, -100， 20190102-154359

这样转账结束后，
- 钱包1
  - RMB, digital, -100, cash, 50
  - USD, digital, -200, cash, 60
  - JPY, digital, -100, cash, 0

- 钱包2，
  - RMB, digital, 100, cash, 20
  - JPY, digital, 100, cash, 0

## 技术需求描述

- 不需要认证
- 不需要高并发
- 实现功能以外的所有细节都可以省略

## 运行效果

- 0， 以HyperLegder来实现简单的示例代码
  - 从一个CSV文件中读取转账指示，发送至全网所有端点
  - CSV文件是可以完全信任的， 时间戳是按照时间排序的
  - CSV文件包含如下栏位： FROM，TO， 资产， 形态， 时间戳

- 1， 能拿到某个时间点(不是当前最新)上的钱包里的余额

```text
例如：取得 2019-08-01-0000时间点的 钱包1的余额
返回：

钱包1
RMB, digital, -100, cash, 50
USD, digital, -200, cash, 60
JPY, digital, -100, cash, 0
```

- 2， 能拿到某个时间段上的相关交易, 

若资产是ALL， 那么返回所有的资产， 如果资产具体指明则仅仅返回该资产

例如钱包2相关的USD资产的交易记录 从t1到t2

返回：

- 钱包1 转给 钱包2  USD, digital, 100， 20181223-120011
- 钱包2 转给 钱包3  USD, cash，   -50， 20181223-120059
